# Meilenstein-Plan für das E-Learning-Tool

1. **Meilenstein: Struktogramm → Code (Listen, Sortieren, Suchen)**
   - **Merkmale:**
      - Aufgabenpaket für Listen/Arrays, Sortier- und Suchalgorithmen.
      - Fokus: Struktogramme lesen und in Quellcode umsetzen.
      - Strikte Nutzung der Operatorenliste (keine Fremdfunktionen).
      - Mindestanforderung: 3 Testfälle pro Aufgabe.
   - **Warum?**: Stärkt das Leseverständnis von Struktogrammen und prüft, ob die Code-Boxen korrekt funktionieren.

2. **Meilenstein: Grundfunktionen und Python-Aufgaben**
   - **Merkmale:**
      - Aufgaben stellen und lösen.
      - Basic Syntax Validation für Python-Code.
      - Backend zur Verarbeitung und Bewertung der Lösungen.
   - **Warum?**: Damit Schüler direkt beginnen können, prüfungstypische Aufgaben zu lösen.

3. **Meilenstein: Struktogramm-Integration**
   - **Merkmale:**
      - Unterstützung für `.stgr`-Dateien:
         - Hochladen, Visualisieren, Bearbeiten.
      - Interaktive Struktogramm-Erstellung für die visuelle Planung von Aufgaben.
      - Struktogramm-Auswertung: Überprüfung auf Logik und Sinnhaftigkeit.
      - Export von Lösungen (Code + Struktogramm).
   - **Warum?**: Struktogramme erleichtern Schülern den Übergang von der Planungs- zur Implementierungsphase und fördern ein tieferes Verständnis.

4. **Meilenstein: Erweiterung auf weitere Sprachen (langfristig)**
   - **Merkmale:**
      - Unterstützung für Java, PHP und JavaScript.
      - Sprachspezifische Syntaxprüfung und Bewertungsmechanismen.
   - **Warum?**: Um das Tool für eine breitere Zielgruppe nutzbar zu machen.

5. **Meilenstein: Erweiterte Funktionen (Gamification und Analyse)**
   - **Merkmale:**
      - Fortschrittstracker: Schüler sehen ihren Lernstand.
      - Gamification: Badges und Belohnungen.
      - Statistiken und Analyse für Lehrer.
   - **Warum?**: Um das Engagement der Schüler zu steigern und Lehrenden Einblick zu geben.
